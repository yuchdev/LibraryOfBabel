From Borges to Meaning:
Progressive Reduction of the Library of Babel

Abstract

This interactive document investigates a sequence of mathematical transformations of The Library of Babel originally proposed by Jorge Luis Borges. Starting from the classical combinatoric construction containing every possible sequence of characters, we progressively constrain the Library so that the number of possible books decreases, average semantic density increases, and random pages become increasingly human-readable.

At each stage, we introduce a mathematical model, combinatoric analysis, and Python algorithms. The experiment reveals a phase transition from pure entropy toward structured language, and eventually toward semantic manifolds approximating meaningful literature.

I. The Hierarchy of Constraints

This section details the mathematical models used to restrict the library. As we apply structural rules—moving from random characters to dictionary words, and then to semantic networks—the combinatorial explosion collapses, leaving a denser concentration of human-readable meaning.

Model 1: The Original Borges Library

Borges defined books with exactly 410 pages, 40 lines per page, and 80 characters per line, utilizing an alphabet of 25 symbols (22 letters, space, comma, period).

Mathematical Model

Total character positions per book:
C = 410 × 40 × 80 = 1,312,000

Total combinatoric volume (Number of books):
V1 = 251,312,000 ≈ 1.95 × 101,834,097

This space maximizes Shannon Entropy: \( H_{max} = -\sum p(x) \log_2 p(x) \approx 4.64 \) bits per character.

# Python Implementation
import random
import string

ALPHABET = string.ascii_lowercase[:22] + " ,."

def generate_borges_page():
    page = ""
    for _ in range(40):
        line = "".join(random.choices(ALPHABET, k=80))
        page += line + "\n"
    return page

Model 2: The Lexical Library

Human language is not arbitrary character sequences. By forcing the library to only select randomly from a dictionary of valid English words, we vastly reduce the combinatorial space while injecting recognizable micro-structures.

Mathematical Model

Assume an active vocabulary of W = 50,000 words. Average word length is ~5 chars + 1 space.

Words per book:
N = 1,312,000 / 6 ≈ 218,666

Total combinatoric volume:
V2 = 50,000218,666 ≈ 101,027,511

Entropy decreases dramatically. Mutual information between adjacent characters increases, but semantic meaning remains near zero.

# Python Implementation
import random

VOCAB = ["time", "light", "shadow", "mirror", "labyrinth", "man", ...] # 50k words

def generate_lexical_page():
    page = ""
    for _ in range(40):
        words = random.choices(VOCAB, k=13) # ~13 words per 80 char line
        page += " ".join(words) + "\n"
    return page

Model 3: Syntactic Markov Chains

To simulate grammar, we apply a First-Order Markov Chain based on a corpus. The probability of the next word depends solely on the current word, establishing local syntactic structure (e.g., nouns often follow adjectives).

Mathematical Model

Transition matrix P where Pij = P(wj | wi).

Because most word pairs are invalid in English grammar, the matrix is highly sparse. Average branching factor (valid next words) drops to k ≈ 100.

Total combinatoric volume:
V3 = 100218,666 ≈ 10437,332

The text begins to resemble schizophrenic poetry. It possesses syntax but lacks long-range semantic coherence.

# Python Implementation
import random

# P_matrix is a dict mapping words to lists of likely following words
P_MATRIX = { "the": ["man", "labyrinth", "mirror"], "man": ["walks", "sees", "is"] }

def generate_markov_page(seed_word):
    page, current_word = "", seed_word
    for _ in range(40 * 13): # words per page
        page += current_word + " "
        current_word = random.choice(P_MATRIX.get(current_word, ["the"]))
    return page

Model 4: Semantic Graph Traversal

Entire books must remain inside a thematic manifold. We construct an ontological graph where nodes are concepts and edges are semantic relationships. A book is a random walk within a specific thematic cluster (e.g., "cosmology", "grief").

Mathematical Model

Graph G = (V, E) partitioned into thematic subgraphs S1, S2....

Branching factor inside a specific sub-topic is severely restricted: k ≈ 15.

Total combinatoric volume per theme:
V4 = 15218,666 ≈ 10257,175

Information-theoretic interpretation: H1 > H2 > H3 > H4 while Mutual Information I(text;meaning) rises sharply. We are approaching the thin manifold of human thought.

# Python Implementation
import random

GRAPH = { # A thematic subgraph for "astronomy"
    "stars": ["burn", "collapse", "shine"],
    "collapse": ["into", "slowly", "violently"],
    "into": ["black_holes", "dust", "void"]
}

def semantic_walk(seed_concept, length):
    result = [seed_concept]
    for _ in range(length - 1):
        next_words = GRAPH.get(result[-1], ["stars"])
        result.append(random.choice(next_words))
    return " ".join(result)

IV. Beyond the Horizon: Progressively Restrictive Futures

To move beyond semantic graphs into true "infinite coherent literature," we must introduce macro-structural constraints. If these steps were added to the research, they would require significant computational leaps.

Step 5: Latent Space Traversal (LLMs)

Instead of discrete word graphs, text generation is modeled as continuous integration over a high-dimensional vector field (Transformer latent space). A book is a smooth trajectory through this space.

Math Model

P(w_t | w_1...w_{t-1}) = softmax(W·h_t)
where trajectory T minimizes semantic drift D(t).

Implementation Needs

Requires loading heavy tensor architectures in memory (e.g., Llama 3) and custom samplers to force deterministic generation from a starting topological coordinate.

Step 6: Global Logical Consistency

The ultimate constraint. A book must not contradict itself from page 1 to 410. The library acts as a formal theorem prover. Character attributes, physics, and causal timelines must resolve perfectly.

Math Model

Let state S_t = S_{t-1} + statement(w_t).
Constraint: Satisfiability(S_t) == True ∀ t

Implementation Needs

Coupling an LLM generator with a symbolic logic engine (like Z3 SMT solver). The text engine proposes next paragraphs; the logic engine verifies facts against the global knowledge base. Computationally immense.
